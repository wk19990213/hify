# Hify 前端 Workbench Pro 设计规格

## 背景

当前前端已具备基础业务能力，但视觉问题明显：

- 多处中文文本显示为乱码，降低专业感与可用性。
- 页面结构偏“入口卡片集合”，不像高频使用的内部工作台。
- 紫蓝渐变使用过重，品牌辨识有余，管理后台的克制感不足。
- 页面级布局重复：列表页标题、描述、按钮、表格工具区各自实现。
- Chat、Workflow、列表页体验割裂，缺少统一壳层和统一操作语义。

目标是将 Hify 调整为面向团队内部的 AI Agent 工作台：清爽、精密、稳定、高效。

## 设计方向

采用 `Workbench Pro`：

- 深色窄侧栏承载一级导航。
- 浅色主工作区承载业务内容。
- 顶部上下文栏展示页面标题、说明、搜索、主操作。
- 内容区优先展示状态摘要、筛选工具条、主表格或编辑器。
- 整体风格参考 Linear、Retool、Supabase 控制台，但避免营销页式大留白。

不采用：

- 全白顶部导航 SaaS 风格：不适合高频管理页。
- 全深色控制台：长时间配置表单、阅读知识库、调试工作流会累。

## 视觉系统

主色从当前紫蓝转为更克制的青绿色：

- 主色：青绿，用于主按钮、选中导航、成功连接、核心动作。
- 强调色：琥珀，用于警告、待配置、需要注意的状态。
- 中性色：石墨黑、Slate 灰、冷白背景。
- 紫色只保留为少量模型/AI 标签辅助色，不作为大面积主视觉。

圆角控制在 6-8px。卡片只用于独立实体、状态摘要、弹窗，不把整页区块套成大卡片。

## 信息架构

全局壳层分三层：

1. 左侧窄导航：Logo、Provider、Agent、Knowledge、Workflow、MCP、Chat、设置/用户。
2. 顶部上下文栏：当前页面名、说明、全局搜索/筛选、主按钮。
3. 主工作区：状态摘要、数据表格、编辑器、聊天窗口。

导航折叠后只显示图标，hover tooltip 显示名称。移动端侧栏变抽屉。

## 页面设计

### 概览页

从“快捷入口卡片”改为“运营概览工作台”：

- 顶部展示平台健康：模型提供商、Agent 数、知识库文档、MCP 工具、最近对话。
- 中部展示最近活动：Provider 健康检查、Agent 更新、Workflow 运行。
- 右侧展示待处理事项：未配置模型、连接失败、未启用 Agent。

### 列表页

Provider、Agent、Knowledge、MCP 使用统一列表模板：

- 顶部上下文栏放标题、说明、主按钮。
- 内容区第一行放状态摘要卡。
- 第二行放筛选工具条：搜索、状态、类型、刷新。
- 主区域为紧凑表格。
- 行操作优先使用图标按钮，保留 tooltip，减少文字按钮拥挤。

### Agent 页面

突出“可运行实体”：

- Agent 名称旁展示状态、模型、工具数、知识库绑定。
- 主要操作为“对话测试”“编辑配置”“禁用/启用”。
- 表单弹窗分区：基础信息、模型参数、系统提示词、知识库、MCP 工具、工作流。

### Provider 页面

突出连接状态：

- 状态摘要：可用、异常、模型总数、最近检测延迟。
- 表格中 Base URL 默认截断，hover 或展开查看完整值。
- 健康状态使用点状状态 + 文案，避免只靠颜色。

### Chat 页面

改成专业调试台：

- 左侧会话列表保留，但更紧凑。
- 中间消息区提升可读性：助手消息宽度更大，代码块更清晰。
- 输入区固定底部，支持 Enter 发送、Shift+Enter 换行。
- 顶部显示当前 Agent、模型、知识库、工具启用状态。

### Workflow 编辑器

改成三栏工作台：

- 左栏：节点库和节点列表。
- 中间：画布，尽量铺满。
- 右栏：选中节点配置、变量提示、边配置。
- 顶部工具栏：返回、保存、运行测试、自动布局、启用状态。

## 组件规范

新增或统一这些基础组件：

- `AppShell`：侧栏、顶部栏、主区布局。
- `PageHeader`：标题、说明、主操作。
- `SummaryMetric`：状态摘要卡。
- `Toolbar`：搜索、筛选、刷新、批量操作。
- `StatusBadge`：统一状态点、标签、语义色。
- `IconAction`：表格行图标操作按钮。
- `EmptyState`：空数据、未配置、加载失败。

优先复用 Element Plus，不引入新 UI 框架。

## 交互规则

- 主操作始终在顶部栏右侧。
- 危险操作必须二次确认。
- 长文本默认截断，提供 hover 或抽屉查看。
- 表格行操作保持一行，不换行挤压。
- 加载、空状态、错误状态都需要统一视觉。

## 实施范围

第一阶段只做视觉与布局升级，不改变后端 API。

包含：

- 修复前端中文乱码。
- 重构全局壳层。
- 更新设计 token。
- 改造概览页、Provider 列表页、Agent 列表页。
- 定义可复用列表页模板和状态组件。

暂不包含：

- 可视化工作流拖拽能力升级。
- 新权限体系。
- 新接口字段。
- 深色模式切换。

## 验证标准

- `npm run build` 通过。
- 主要页面无乱码。
- 1366px 桌面下列表页一屏可见标题、摘要、工具条、表格首屏。
- 375px 移动端侧栏不遮挡内容，主按钮不溢出。
- 文本不重叠，按钮文字不截断。
- 页面不再呈现单一紫蓝主题。
